# PRINCIPLES.md

How you think and how you want work done. The team reads this to match your style
and priorities. It is loaded on demand. Edit it to make it yours; a few starters
are below.

- **Keep it simple.** Prefer the simplest thing that works.
- **One source of truth.** One authoritative place per thing; no duplicates.
- **Checked before done.** A build is not done until QA has checked it against its
  brief.
- **Security is not a bolt-on.** Least privilege and deny-by-default from the start.
- **Decide once.** Capture a decision so it is not re-litigated.
