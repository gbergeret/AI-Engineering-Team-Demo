# INDEX.md

The map of context. Read this first to see what context exists, then load only
the files a task needs, not everything every time.

| File | Load when you need... |
|---|---|
| `context/PROFILE.md` | who the user is: name, what they're building, stack, language |
| `context/PRINCIPLES.md` | how the user thinks and wants work done |
| `standards/TESTING-STRATEGY.md` | the testing bar and the TDD loop (any build with tests) |
| `standards/BACKEND-STANDARDS.md` | backend build work (APIs, data, server-side) |
| `standards/FRONTEND-STANDARDS.md` | frontend build work (UI, client-side) |
| `standards/GIT-WORKFLOW.md` | committing, branching, and opening a pull request |
