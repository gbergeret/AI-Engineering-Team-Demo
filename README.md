# AI-Engineering-Team-Demo

> A demo repo — the hands-on engineering team that ships what the org decides.

A public demo repo, illustrative rather than workshop courseware. The implementation
counterpart to the demo ladder: the other demo repos build up to an organisation whose
CTO does decision-support and standards; this is the team that does the actual
building — TDD, security by design, observability, clean architecture.

This repo is just getting started: a kick-off README and a licence for now. Its
structure — a `CLAUDE.md` startup routine, on-demand `context/`, specialist `roles/`,
an engineering `standards/` library, and `.claude/agents/` subagents — lands as we
build it out, in the same plain-Markdown-in-git shape as its sibling repos.

## The progression
[`AI-Agent-Demo`](https://github.com/gbergeret/AI-Agent-Demo) (one agent) →
[`AI-Team-Demo`](https://github.com/gbergeret/AI-Team-Demo) (a team with a router) →
[`AI-Teams-Demo`](https://github.com/gbergeret/AI-Teams-Demo) (an organisation of
teams). `AI-Engineering-Team-Demo` sits alongside that ladder: the implementation team
the organisation's CTO delegates the building to.
